# ✈️ Flight-point

**Flight-point** is a premium, high-fidelity flight search and booking platform built with React, Vite, and GSAP. It features a modern, responsive design with smooth animations and a sophisticated user interface tailored for a seamless travel planning experience.

## 🌟 Key Features

-   **Dynamic Hero Section**: Interactive reveal animations that transition from a landing state to an active booking interface.
-   **Advanced Flight Search**: Comprehensive search functionality including departure/arrival locations, date selection, traveler counts, and cabin classes.
-   **Premium UI/UX**: Built with a "wow" factor in mind, utilizing modern typography, glassmorphism, and curated color palettes.
-   **Smooth Animations**: Leverages GSAP for high-performance, fluid transitions and micro-interactions.
-   **Mobile Optimized**: Responsive layouts including a "Continue in App" mobile prompt and responsive navigation.
-   **Comprehensive Sections**: Includes Announcement Bar, Interactive Navbar, Pricing Plans, Explore destinations, and a detailed About section.
-   **Smart Loading**: Custom loading screen with session-based persistence to enhance perceived performance.

## 🛠️ Technology Stack

-   **Frontend**: [React 19](https://react.dev/)
-   **Build Tool**: [Vite 8](https://vitejs.dev/)
-   **Styling**: [Tailwind CSS 4](https://tailwindcss.com/)
-   **Animations**: [GSAP (GreenSock Animation Platform)](https://greensock.com/gsap/)
-   **Icons**: [Lucide React](https://lucide.dev/)
-   **Components**: [Radix UI](https://www.radix-ui.com/)
-   **Utilities**: `clsx`, `tailwind-merge`, `date-fns`
-   **Form Elements**: `react-datepicker`

## 🚀 Getting Started

### Prerequisites

-   Node.js (Latest LTS recommended)
-   npm or yarn

### Installation

1.  Clone the repository:
    ```bash
    git clone <repository-url>
    ```

2.  Install dependencies:
    ```bash
    npm install
    ```

3.  Start the development server:
    ```bash
    npm run dev
    ```

### Building for Production

To create an optimized production build:
```bash
npm run build
```

## 📂 Project Structure

-   `src/components/`: Modular React components (Hero, Navbar, Search, etc.)
-   `src/components/ui/`: Reusable UI primitives.
-   `src/assets/`: Static assets like images and logos.
-   `src/data/`: Mock data or configuration constants.
-   `src/lib/`: Utility functions and library configurations.
-   `src/index.css`: Global styles and Tailwind configuration.

Flight-point prioritizes a **premium aesthetic**. This includes:
